\begin{abstract}
This report presents an end-to-end NLP project for review understanding on Amazon Gift Card data. The current pipeline has three operational layers: sentiment classification with uncertainty control, multi-label issue extraction, and transformer-based contextual extension. The source dataset has 152,410 reviews, while issue annotations contain 19,000 merged and validated rows. The selected classic sentiment variant (V6: word plus character TF-IDF with negation tagging) reaches test precision\_0 of 0.677, recall\_0 of 0.932, and F2\_0 of 0.867. The transformer extension reaches test precision\_0 of 0.822, recall\_0 of 0.833, and F2\_0 of 0.831 with 0.910 coverage at threshold 0.40/0.60. The issue extractor reaches test micro-F1 of 0.765 for the word model and 0.763 for the word plus character model, with better macro-F1 in the latter setup. In the current Level-3 upgrade, the issue module adds per-label class-weight search, probability calibration, and threshold-stability diagnostics. A new commandized classic ablation block (negation/char/lexicon toggles) is also added; in the latest run, V7 gives the strongest negative-first trade-off in that block (test recall\_0=0.869, precision\_0=0.636, F2\_0=0.810). To align tightly with the NLP syllabus, the project also includes a benchmark suite (Logistic Regression, Naive Bayes variants, Perceptron, LinearSVM, SGD log-loss, SVD-semantic logistic, FFNN), an N-gram language-model block, an RNN/LSTM baseline, an MLM probe artifact, and an LLM prompt-style baseline. The updated course-fit matrix computed from generated artifacts reports estimated syllabus coverage of 94.3\%.
\end{abstract}

\section{Introduction}
\subsection{Context}
Natural language in user reviews is noisy, short, and mixed. A single review can include praise, complaint, slang, typo, and domain words in the same sentence. This makes review understanding a practical NLP benchmark where both classic and modern methods can be evaluated under a common objective \cite{jurafsky2026,turney2002}.

\subsection{Problem Statement}
This study focuses on Amazon Gift Card reviews. The objective is not limited to positive-versus-negative sentiment discrimination; it is to construct a review-understanding pipeline that supports uncertainty-aware decisions and issue-level routing for downstream service operations.

\subsection{Objectives and Scope}
The project objectives are:
\begin{enumerate}
\item Build a high-recall, risk-aware sentiment classifier for negative feedback triage.
\item Extract concrete issue categories using multi-label text classification.
\item Compare classic sparse models with transformer-based contextual modeling.
\item Expand artifact coverage to match core and advanced NLP course topics.
\item Keep all outputs reproducible from commands and saved metrics files.
\end{enumerate}

\subsection{Implemented System Scope}
The implementation is structured as an end-to-end NLP pipeline. It includes text normalization, lexical and subword feature modeling, negation and contrast handling, supervised training, uncertainty control, issue extraction, transformer extension, benchmark synthesis, and demonstration integration.

\subsection{Main Contributions}
The main contributions of the current repository state are:
\begin{enumerate}
\item A reproducible sentiment pipeline with high class-0 recall and explicit threshold trade-off analysis.
\item A multi-label issue extraction module with per-label threshold tuning and per-label metrics.
\item A transformer extension with selective prediction metrics and hard-case evaluation.
\item A syllabus-upgrade package covering NB, Perceptron, FFNN, N-gram LM, RNN/LSTM, MLM probe, and LLM prompt baseline.
\item A Level-3 classic upgrade package: per-label class-weight search, probability calibration, threshold-stability diagnostics, and commandized ablation outputs.
\item A documentation set including contribution matrix, quality-gate template, scoreboard, tests, and demonstration scripts.
\end{enumerate}

\section{Background and Related Works}
\subsection{Sentiment and Text Classification Baselines}
Classic sentiment models using sparse n-gram features and linear classifiers remain well-established baselines for noisy user-generated text \cite{wang2012,manning2008ir}. They are computationally efficient, interpretable through feature weights, and suitable for threshold-based risk control.

\subsection{Negation and Contrast in Sentiment}
Negation scope is a known challenge in sentiment analysis \cite{wiegand2010}. In review text, patterns like \textit{not bad} and contrast forms such as \textit{good but late delivery} can flip decision boundaries. This motivates explicit negation tagging and clause-aware feature design.

\subsection{Multi-label Learning for Issue Tagging}
Issue extraction is a multi-label problem because one review can mention multiple complaint categories \cite{tsoumakas2011,zhang2014}. One-vs-rest setups with per-label threshold calibration are practical for imbalanced label distributions.

\subsection{Contextual Modeling and Transformer Extensions}
Transformer-based encoders capture contextual semantics and can improve robustness on idiomatic short text \cite{vaswani2017,devlin2019}. However, in risk-aware settings, selective thresholds and business-specific objective trade-offs remain essential.

\section{NLP Problem Definition}
\subsection{Task A: Sentiment Understanding}
Task A is binary sentiment classification from review text. Ratings 1 and 2 are mapped to negative class. Ratings 4 and 5 are mapped to positive class. Ratings with value 3 are used for uncertainty analysis and are excluded from the high-confidence supervised training set \cite{turney2002,wang2012}.

The decision policy is not accuracy-first. In triage settings, false negatives on adverse reviews are costly. Consequently, model selection follows a negative-first heuristic: prioritize recall for class 0, then F2 for class 0, then precision for class 0.

\subsection{Task B: Issue Extraction}
Task B is multi-label issue extraction. One review can contain more than one issue, so single-label tagging is not enough \cite{tsoumakas2011,zhang2014}. The output label set in this project has nine issue types and allows multi-label assignment per row.

This task maps unstructured review text to structured issue signals. It enables transition from generic sentiment outputs to operational action categories.

\subsection{Task C: Deep Context Extension}
Task C is transformer fine-tuning as NLP extension. The extension aims to test deep contextual language modeling for difficult short phrases and negation patterns. It also provides a direct comparison with classic sparse representations \cite{vaswani2017,devlin2019}.

\section{Text Processing Design}
\subsection{Normalization}
The core normalization pipeline uses lowercase conversion, URL removal, punctuation cleanup, and white-space normalization. This reduces token sparsity while keeping core sentiment words.

Optional abbreviation normalization maps common short forms to full forms. This includes mappings such as gr8 to great, thx to thanks, u to you, and idk to i do not know.

\subsection{Negation Handling}
Negation is a key NLP challenge in short reviews. The project uses a negation tagging method. After a negation cue, the next few tokens receive a NOT\_ prefix. The window size is three tokens by default \cite{wiegand2010}.

This method is computationally lightweight and effective for sparse models. It helps separate good from NOT\_good and bad from NOT\_bad in feature space.

\subsection{Contrast Handling}
The pipeline also models simple discourse contrast. When markers such as but, however, although, though, and yet appear, text can be split into left and right clauses. Separate clause features and marker flags are then added.

This design captures a common review pattern where the second clause carries the final user opinion.

\subsection{Character-level Modeling}
Character 3 to 5 gram features are used as optional union with word features. This helps with typo, short slang, and small spelling variants. In NLP tasks on user text, this often improves robustness \cite{joulin2017}.

The project includes an AFINN-based sentiment lexicon block in one variant. It computes raw score, negation-aware score, intensity-aware score, and contrast-weighted score. This adds explicit lexical prior to sparse representation \cite{nielsen2011}.

This part is useful for NLP course learning because it connects classic lexicon methods with modern supervised learning in one pipeline.

\section{Modeling Approach}
\subsection{Sentiment Classifiers}
The main sentiment classifier is Logistic Regression. Chi-square feature selection is applied with top K values from 1,000 to 10,000. Class weight settings include none, balanced, and weighted variants \cite{forman2003}.

Additional sentiment models include L1 Logistic Regression for sparse embedded behavior, Decision Tree, and Random Forest.

\subsection{Issue Extraction Model}
Issue extraction uses one-vs-rest training across nine labels. The default model is Logistic Regression with balanced class weight. A LinearSVM baseline is also available \cite{rifkin2004}.

In the current code state, the issue module also supports per-label class-weight search over \{none, balanced, w2, w5, w10\}, optional probability calibration (sigmoid or isotonic), and explicit threshold-stability reporting under controlled threshold perturbations.

Threshold tuning is done per label on validation data. This is important in multi-label NLP because one global threshold is often not enough for all labels.

\subsection{Transformer Extension}
The transformer extension fine-tunes a sequence classifier with class-weighted loss. It uses cleaned text and evaluates with the same uncertainty and threshold framework for direct comparison \cite{devlin2019}.

This extension gives deeper contextual semantics and helps analyze hard short phrases where sparse models may fail.

\section{Experimental Setup}
\subsection{Split and Protocol}
All experiments use seed 42. Strong sentiment labels are split with train, validation, and test ratio 70, 10, and 20. Three-star reviews are held for uncertainty analysis.

Issue extraction uses 19,000 labeled rows with split sizes 13,300 for train, 1,900 for validation, and 3,800 for test in saved runs.

\subsection{Main Metrics}
For sentiment, key metrics are precision\_0, recall\_0, F1\_0, and F2\_0, along with full accuracy and F1. Because the strong-label set is imbalanced, model trade-offs are interpreted mainly through precision-recall behavior rather than ROC-only views \cite{saito2015}. Selective metrics under uncertainty include coverage, uncertain rate, selective precision\_0, selective recall\_0, selective F2\_0, and false-negative rate on covered samples.

For issue extraction, key metrics are micro-F1, macro-F1, subset accuracy, hamming loss, and per-label precision, recall, and F1.

\subsection{Uncertainty Rule}
The uncertainty policy uses low threshold and high threshold on positive probability. It follows a selective classification abstention setup and can return uncertain in the middle range \cite{geifman2017}. We apply an additional heuristic in the classic pipeline that returns uncertain for too short or sparse text cases.

This policy improves deployment reliability because low-information text is not forced into a hard class decision.

\subsection{Cross-understanding, Review, and Reproducibility}
The workflow is designed so that each critical module can be executed by at least one member who is not the original author. For each module handover, the receiving member reruns the pipeline from saved commands and checks three gates: no train-test leakage, fixed random seed behavior, and consistency between table values and exported CSV files.

For code quality control, we use a branch-and-review policy with one non-author review per substantial change. The review checklist focuses on metric correctness, split integrity, and reproducibility. This process aligns with course requirements on module split, shared understanding, and cross-review practice. The latest issue-module update also enforces natural ID ordering for annotation templates and validates this behavior through dedicated regression tests.

To make teamwork evidence auditable in the final presentation, the evidence package includes: role/module matrix, command logs for reruns, model-output CSV artifacts, and review notes linked to changed modules. This documentation complements quantitative model results and substantiates process quality in addition to final scores.

\section{No-New-Data Syllabus Upgrade Package}
The project now includes an extra package that improves alignment with NLP course topics without adding any new labeled data. The package reuses the same train, validation, and test split, and keeps the same text cleaning policy.

The first upgrade block is a comprehensive classic benchmark suite. It includes Logistic Regression, Multinomial Naive Bayes, Complement Naive Bayes, Perceptron, LinearSVM, SGD with log-loss, semantic Logistic Regression on SVD vectors, and a feed-forward MLP on the same SVD vectors. The training size is capped at 60,000 rows in this block to control runtime while preserving class ratio.

The second upgrade block is a language model block. It trains unigram and bigram language models with add-k smoothing and reports perplexity on validation and test splits. This gives direct evidence for the N-gram language model topic in the course \cite{jurafsky2026,manning2008ir}.

The third upgrade block adds sequence and modern-language-model artifacts, including an RNN/LSTM sentiment baseline, an MLM probe report, and an LLM prompt-style semantic baseline on the same sentiment split.

The final block builds a topic-coverage matrix from project artifacts. The matrix uses explicit topic scores and reports an overall syllabus coverage score for defense discussion.

\subsection{Threshold Strategy in Multi-label Setting}
Issue extraction uses per-label tuned thresholds. Threshold profiles differ between word-only and word+char setups because score distributions change by label.

This behavior is expected in NLP multi-label tasks where each label has different prior frequency and text signal strength.

\subsection{Template Generation Reproducibility}
The issue-label template builder now applies a natural ID sort before CSV export. The implemented key orders numeric identifiers by integer value, then non-numeric identifiers by case-insensitive lexical order, and finally empty identifiers. This avoids lexicographic artifacts such as \texttt{10} being placed before \texttt{2}.

Two regression tests are included in \texttt{tests/test\_issue\_template\_sort.py}: one unit test for the sort key and one end-to-end test for output ordering from \texttt{cmd\_make\_template} in \texttt{src/issue\_steps/steps.py}.

This update improves annotation reproducibility and team handoff reliability, while leaving the model-evaluation metrics reported above unchanged.

\section{Classic Versus Transformer Discussion}
Classic sparse NLP and transformer-based NLP exhibit distinct performance profiles. In this setting, classic V6 maintains high negative recall and high selective recall, whereas the transformer model provides higher overall accuracy and higher negative precision.

For triage scenarios prioritizing detection of adverse reviews, classic V6 is more closely aligned with the decision policy. For semantic nuance and idiomatic phrasing, transformer outputs provide complementary value when combined with threshold control.

A hybrid strategy is therefore justified: use the classic model as the primary triage layer and apply transformer scoring as a secondary analysis step for ambiguous cases.

\section{Academic Relevance to the NLP Course}
This project now shows a wide set of NLP ideas in one system. It covers text normalization, lexical mapping, n-gram sparse encoding, negation scope handling, clause-level cues, multi-label text classification, threshold calibration, and transformer fine-tuning. It also includes explicit Naive Bayes, Perceptron, feed-forward semantic baselines, N-gram language model artifacts, an RNN/LSTM baseline, an MLM probe, and an LLM prompt-style application artifact.

The project remains directly usable for course-based experimentation. It provides saved artifacts, reproducible metrics, annotation workflow records, and deployable inference scripts. It also includes a topic-coverage matrix suitable for oral defense.

For course learning, the updated version provides stronger evidence of trade-offs between classic and deep NLP approaches under one business objective and one fixed dataset.

\section{Limitations and Future Work}
The main limitation is still data imbalance, especially in negative sentiment and rare issue labels. This affects precision and stability in several models.

Another limitation is uneven depth across topics. Although coverage is high, some topics are represented by lightweight artifacts (for example regular expressions at 50\% coverage and compact MLM/LLM probes).

Future work can focus on quality depth, not only topic presence. Since commandized ablation is now available, the next steps are richer error clustering for false negatives, stronger regex-focused preprocessing analysis, better calibration diagnostics under distribution shift, and tighter demo instrumentation linking each prediction to its contributing module.

\section{Conclusion}
This work presents a complete review-understanding pipeline spanning sentiment classification, issue extraction, and transformer-based extension. The classic context-aware sparse model provides high negative-class recall under the negative-first policy, while the transformer extension improves precision and contextual robustness on selected cases. Multi-label issue extraction transforms sentiment output into operationally relevant issue categories. The syllabus extension block covers classic baselines, N-gram language models, RNN/LSTM, MLM probing, and LLM prompt artifacts without introducing new labeled data. With the current artifact set, the project demonstrates reproducibility and an estimated syllabus coverage of 94.3\%.
